Scope

Since Migrating all Communication, messaging, collaboration and content /document management workloads to the Microsoft 365
Cloud platform four months ago, DreamTal corporation has been able to stop the leaks of confidential information that was plaguing
the sales team previously. Since the customer started using Microsoft 365, Employee productivity has increased by 500% when compared
with this time last year – thanks to Microsoft Team. The company has also implemented a streamlined records management and basic information
security system using Azure and SharePoint.

The new financial year just started for DreamTal Corporation, and the Company has budgeted some money for expanding the current deployment of
Microsoft 365 cloud applications, which includes SharePoint Online, Exchange Online and Microsoft Teams.

Although, DreamTal has been using these cloud services for almost one year, it was only available to executives and members of the sales team, the
new requirement is for Microsoft 365 to be deployed across the entire company by the end of the Q1, in the coming calendar year.

DreamTal Corporation expects this project to be completed in the next 3 weeks.

Ensure that you add one Member of each group as group Owner, except for the “Facilities department group”.
In the Group expiration settings, ensure you specify the “email” address of the main administrator as the “Contact email” in the “Email contact for
groups with no owners”
Configuring Microsoft Teams for all departments.
Create Microsoft Teams from existing groups: Craete a Microsoft Team for each of Department using the Azure groups you already created for
each team – except for “Human resources” group. Meaning, you should create the Team from the “Microsoft 365 Admin Center” page –
under Active groups, not from the “Teams Admin Center” page.
Create Microsoft Teams from scratch: Create the Team for the “Human resources” department from the Teams Admin center.
Configuring Microsoft Teams Meeting Policies:
Central Admin Department: Meeting Policy for this group will must ensure the following:
Allow Transcription
Allow Cloud Recording
Allow Private meetings
Allow channel Meeting scheduling.
Note: You should assign this meeting policy to the “Central Admin group” not to individuals.

Frontline Support Department: Meeting Policy for this group will must ensure the following:
Allow Cloud Recording
Allow Private meetings
Allow channel Meeting scheduling.
Note: You should assign this meeting policy to the “frontline support group” not to individuals.

Facilities Department: Meeting Policy for this group will must ensure the following:
Allow Transcription
Allow Cloud Recording
Note: You should assign this meeting policy to the “facilities group” not to individuals.

Finance and Accounting Department: Meeting Policy for this group will must ensure the following:
Do not let Anonymous people start a meeting.
Ensure that only people in your organization(Dreamtal employees) are automatically admitted into meetings.
Disable Live captions.
Note: You should assign this meeting policy to the “Finance and Accounting group” not to individuals.

Human Resources: Meeting Policy for this group will must ensure the following:
Do not let Anonymous people start a meeting.
Ensure that only people in your organization(Dreamtal employees) are automatically admitted into meetings.
Note: You should assign this meeting policy to the “Human Resources group” not to individuals.

Configuring Sharepoint sites and document repositories for all departments: Create one sharepoint site/ document library for each department.
Sharepoint site for each department should be such that can be used to store documents and enable collaboration for Members of the site(department)
Assign one member of each department as the “Site owner”, then make the second just a “member”.
Configuring sharepoint Sharing and Access control Policies: Configure site sharing policies for site site.
Ensure that content of each department’s sharepoint site can only be shared with people in Dreamtal(Organization).
Don’t do this globally, do it for each site individually.
For Acccess control settings, ensure that full access is allowed for “Unmannged devices”
which are devices that are not enrolled to DreamTals device management solution.
Configuring sharepoint site settings for DreamTal Tenant:
For site settings, ensure that “Default Admin center” is set to the “New sharepoint Admin center”
Configuring Microsoft Exchange recipient management(Shared mailbox, Send-As and Send-on-behalf, Equipment Mailboxes)
Shared mailbox
Create One shared mailbox for the Facilities department, and ensure that all members of the facilities department have access to this mailbox.
Create One Quipment mailbox for “DreamTal equipments”
Create One Room mailbox for “Human resource conf room”
Configuring Microsoft 365 Retention Policies: Create Retention policies for each department with the following settings:
Central Admin Department: Assign to the Central admin group.
Retain for 7 years
Delete items automatically after retention period is over
Frontline Support Department: Assign to Teams channels
Retain for 5 years
Delete items automatically after retention period is over
Facilities Department: Assign to shared Mailbox created for this department.
Retain for ever
Finance and Accounting Department: Assign to Teams channels
Retain for ever
Human Resources: Assign to Sharepoint site
Do not retain, but delete when the content in this repository reach certain age – which should be 7 years
Consider deletion based on when it was created.
Configure Exchange Admin roles for sub Admins:
You need to ensure that members of the Frontline support department are able to “to create, manage, and remove Exchange
recipient objects such as mailboxes in the DreamTal exchange Online organization”.
